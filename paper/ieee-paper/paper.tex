\begin{abstract}
Automated digitization of scanned analog circuit diagrams into simulation-ready SPICE netlists remains a challenging problem in document understanding. The critical bottleneck is not component detection or wire extraction, but the \emph{joining} step---determining which extracted wire segments connect to which component pins and to each other. Existing approaches use radius-based proximity grabbing, which over-merges components onto shared nets, particularly under the noisy endpoint estimates produced by real detectors. We present (1)~an endpoint-graph join model that represents both wire endpoints and component pins as graph nodes with four edge types capturing wire-body, endpoint-endpoint, endpoint-pin, and T-junction connectivity; (2)~a degree-budget completion algorithm that uses min-cost b-matching to recover dropped connections while structurally bounding over-merge through a per-pin edge budget and self-loop guards; and (3)~a synthetic evaluation framework with controlled error injection across five severity levels, enabling robustness analysis independent of ground-truth wire labels. On a benchmark of 134 scanned circuit images from the CGHD-1152 dataset, our pipeline achieves wire detection F1~=~0.976 and the degree-budget strategy maintains join F1~=~0.94 under synthetic error injection at the highest severity level, compared to F1~=~0.36 for the production baseline. The full pipeline generates SPICE netlists that simulate to the exact ground-truth operating point under clean detection, and produce simulation-accurate source currents for up to 62\% of circuits (mean 33\% across the synthetic suite) even at the highest noise level.
\end{abstract}

\section{Introduction}

Analog circuit schematics encode functional information---component types, values, and topological connections---in a visual format designed for human interpretation. Converting scanned schematics into machine-readable SPICE netlists enables automated simulation, design verification, and large-scale circuit analysis. While recent advances in object detection have made component recognition increasingly reliable, the \emph{joining} problem---determining which wire segments connect to which component pins and to each other---remains the primary failure mode in end-to-end digitization pipelines.

The challenge is threefold. First, wire endpoints extracted from binarized images are inherently noisy: the same circuit drawn by different drafters produces endpoint positions that vary by 10--80 pixels. Second, components occupy bounding boxes that overlap with wire endpoints, creating ambiguity about whether an endpoint connects to a component's pin or merely passes nearby. Third, T-junctions, collinear fragments, and crossing wires create complex connectivity patterns that simple radius-based approaches cannot resolve without over-merging.

A natural alternative is to prompt a large vision-language model (VLM) to read the schematic and emit its netlist directly. In our experiments, a strong VLM reliably recognizes component types and the overall layout but does not track which terminals are electrically common---it omits 26--67\% of the wire connections and returns free-form text rather than a structured, simulatable netlist.
This motivates an explicit, inspectable geometric pipeline whose output is a netlist by construction.

We address these challenges with a two-stage approach: (1)~an \emph{endpoint-graph join} model that constructs a graph over both wire endpoints and component pins, with four edge types capturing distinct connectivity patterns; and (2)~a \emph{degree-budget completion} algorithm that identifies and reconnects dropped connections using constrained optimization. We evaluate on the CGHD-1152 dataset~\cite{bayer2021cghd} with a synthetic error injection framework that enables systematic robustness analysis.

\section{Related Work}

\subsection{Circuit Diagram Understanding}
Kelly and Cole~\cite{kelly2024digitizing} presented a pattern-recognition workflow for digitizing circuit schematics with 86.4\% success rate, combining OCR for component values with network graph construction. Kulkarni et al.~\cite{kulkarni2025enhancing} proposed Near Sight Correction (NSC) using Vision-Language Models, achieving 87.4\% accuracy on circuit VQA and 0.87 F1 on connection matrix prediction. These works focus on VLM-based understanding; our approach targets the geometric joining problem that precedes semantic interpretation.

\subsection{Wire Detection and Joining}
Classical approaches extract wires via edge detection, morphological operations, and Hough transforms~\cite{alam2022survey}. The production join in existing pipelines uses radius-based proximity: each wire endpoint grabs all component pins within a fixed threshold (typically 30px), connected via transitive union-find. This over-merges when multiple components fall within the threshold, shorting two-terminal parts (R, C, L, D) onto the same net. Our endpoint-graph model resolves this by incorporating component geometry and directional preferences into the join decision.

\subsection{Graph-Based Connectivity}
Union-find structures are standard for connected-component labeling in image processing. Recent work on graph neural networks for circuit analysis~\cite{alrahis2021gnn} focuses on learned representations for design optimization. Our work uses graph connectivity as a \emph{geometric} model---edges encode spatial relationships between wire endpoints and component pins---rather than a learned representation.

\section{Method}

\subsection{Pipeline Overview}

The full pipeline proceeds through six stages: (1)~component detection via oriented bounding box (OBB) YOLO model; (2)~component occlusion (filling detected regions with local median color); (3)~Sauvola adaptive binarization; (4)~connected component labeling with morphological post-processing; (5)~wire endpoint extraction via PCA on endpoint regions; and (6)~wire joining via the endpoint-graph model with degree-budget completion. The output is a netlist mapping component pins to electrical nodes, suitable for SPICE netlist generation.

\subsection{Endpoint-Graph Join Model}

Given a set of wire segments $\mathcal{W} = \{w_1, \ldots, w_m\}$ with endpoints $\text{ep}(w_i) = (a_i, b_i)$ and component pins $\mathcal{P} = \{p_1, \ldots, p_n\}$, we construct an undirected graph $G = (V, E)$ where $V = \text{eps}(\mathcal{W}) \cup \mathcal{P}$.

\textbf{Edge type 1 (wire body):} For each wire $w_i$, add edge $(a_i, b_i)$ with weight 0. This ensures both endpoints of a wire are always in the same net.

\textbf{Edge type 2 (endpoint--endpoint):} For endpoint pairs $(e_i, e_j)$ with $\|e_i - e_j\| \leq \tau_{\text{join}}$, add edge with weight $\|e_i - e_j\|$. This captures wire fragments, corners, and junctions.

\textbf{Edge type 3 (endpoint--pin):} For each endpoint $e$ and pin $p$ with $\|e - p\| \leq \tau_{\text{pin}}$, add edge with weight $\|e - p\| \cdot (1 - \alpha \cdot \cos\theta)$, where $\theta$ is the angle between the wire direction and the endpoint-to-pin vector, and $\alpha$ controls directional preference. This is the critical edge type: it connects wires to components.

\textbf{Edge type 4 (endpoint--wire-body):} For each endpoint $e$ and wire segment $w_j = (c, d)$, if the point-to-segment distance $\text{dist}(e, w_j) \leq \tau_t$, add edge with weight $\text{dist}(e, w_j)$. This captures T-junctions where a wire terminates on another wire's body.

The tolerances $\tau_{\text{join}}$, $\tau_{\text{pin}}$, and $\tau_t$ are \emph{scale-relative}: $\tau = k \cdot s$ where $s$ is the characteristic scale (median component diagonal), enabling a single parameter setting across circuits of varying size.

\textbf{Component-first pin assignment:} For edge type 3, we use a two-step assignment. First, assign each endpoint to its nearest component by bounding-box distance (0 if inside the bbox, using a generous radius of $\max(\tau_{\text{pin}}, 0.5 \cdot \text{diagonal})$). Second, within the assigned component, select the pin by geometric position: for horizontal components ($w > h$), left pin if endpoint $x <$ center $x$, else right pin; for vertical components ($h > w$), top pin if endpoint $y <$ center $y$, else bottom pin. This handles the common case where endpoints land inside a component's bounding box but far from its pin.

Nets are computed as connected components of $G$, projected onto pins. The result is a \texttt{Netlist} mapping each pin to a net identifier.

\subsection{Degree-Budget Completion}

The endpoint-graph join may leave some component pins ``floating''---touching only their own component's net, with no wire connecting them to another component. This indicates a dropped or over-displaced wire endpoint that the detector failed to place near the correct pin.

Degree-budget completion identifies these floating pins and reconnects them via constrained optimization:

\begin{enumerate}
\item \textbf{Identify floating pins:} A pin $p$ is floating if its net contains only pins from $p$'s own component (excluding inert types: gnd, vss, text, antenna, junction, terminal).

\item \textbf{Candidate edges:} For each floating pin $p$, generate candidate edges to all pins $q$ in other components within reach $r = \text{REACH\_FACTOR} \cdot s$ (default REACH\_FACTOR~=~2.5). Edge weight is Euclidean distance $\|p - q\|$.

\item \textbf{Min-cost b-matching:} Solve $\min \sum_{(p,q) \in E'} w(p,q) \cdot x_{pq}$ subject to $\sum_q x_{pq} \leq 1$ for each floating pin $p$ (at most one connection per pin), using the Hungarian algorithm~\cite{kuhn1955} (\texttt{scipy.optimize.linear\_sum\_assignment}). Each target pin offers up to three slots so legitimate multi-terminal nets are not starved, and a per-pin opt-out leaves a pin with no good target unconnected rather than forcing a spurious edge.

\item \textbf{Self-loop guard:} Reject any matching edge $(p, q)$ if $p$ and $q$ belong to the same component or if adding the edge would merge two nets that already share a component. This structurally prevents short-circuiting two-terminal parts.
\end{enumerate}

Completion edges are ``wireless''---they represent inferred connections without corresponding wire segments, analogous to manual pin merges in schematic editors.

\section{Synthetic Evaluation Framework}

\subsection{Circuit Corpus}

We author 15 synthetic circuits spanning 3--6 components and 2--6 nets, including parallel resistors, series dividers, Wheatstone bridges, RC/RL circuits, diode loops, and a 6-component ring (the hardest topology for join robustness). Each circuit has a known ground-truth netlist and SPICE deck with a test voltage source.

\subsection{Error Injection Model}

We inject five categories of error at four severity levels (L1--L4) on top of the clean (L0) baseline:

\begin{itemize}
\item \textbf{Jitter:} Gaussian endpoint perturbation with standard deviation $\sigma$ (L1:~3px, L4:~14px)
\item \textbf{Cut-short:} Each wire shortened inward by a fixed length (L1:~4px, L4:~22px), modeling endpoints that stop short of the pin
\item \textbf{Drops:} Entire wire removed with probability $p_{\text{drop}}$ (L1:~0, L4:~0.20)
\item \textbf{Wrong-pin:} Endpoint snapped to a wrong nearby pin with probability $p_{\text{wp}}$ (L2:~0.02, L4:~0.10)
\item \textbf{Anchor displacement:} With probability $p_{\text{disp}}$ (L1:~0.10, L4:~0.40), one endpoint is displaced by up to $a_{\max}$ pixels (L1:~30px, L4:~80px)
\end{itemize}

\subsection{Metrics}

\textbf{Join F1:} Component-connectivity F1 score measuring which pairs of components share a net in the predicted vs.\ ground-truth netlist.

\textbf{Simulation accuracy (sim\_ok):} Percentage of random seeds where every source current in the predicted SPICE netlist matches the clean oracle within 5\%.

\section{Discussion}

\textbf{Why graph-based joining works.} The endpoint-graph model resolves ambiguity that radius-based methods cannot: by representing both wire endpoints and component pins as nodes with typed edges, it distinguishes between a wire that \emph{passes near} a component (endpoint--wire-body edge) and one that \emph{connects to} it (endpoint--pin edge with directional preference). The component-first assignment further disambiguates by leveraging the geometric constraint that wires must reach a component's pin, not just its bounding box.

\textbf{Completion as fault recovery.} Degree-budget completion addresses the fundamental limitation of detector noise: even a 97.6\% accurate wire detector drops $\sim$2\% of wire endpoints, and those dropped endpoints are precisely the connections that matter most for simulation accuracy. The b-matching formulation bounds the damage from mis-detection---each floating pin gets at most one recovery edge---while the self-loop guard prevents the catastrophic short-circuit failures that plague unconstrained approaches.

\textbf{Limitations.} The synthetic error model is a first-pass approximation, not yet calibrated to the real detector's measured failure distribution, so the synthetic join numbers are robustness and regression signals rather than predictors of real-image performance. The evaluation draws from a single hand-drawn corpus (CGHD-1152); generalization across other drawing styles, capture conditions, and printed schematics remains to be validated. Finally, the real connectivity metric is structural---it cannot certify that a recovered connection is the correct one, since the corpus has no net-level ground truth---which is precisely what motivates the synthetic framework. Component detection at 88.5\% mAP50 also leaves room for improvement, particularly on crossovers (70.7\% recall).

\section{Conclusion}

We present a two-stage wire joining approach for circuit diagram digitization that dramatically improves robustness under detector noise. The endpoint-graph join model, combined with degree-budget completion, maintains F1~=~0.94 under extreme synthetic error injection, compared to 0.36 for production baselines. The synthetic evaluation framework provides a principled way to analyze join robustness independent of ground-truth wire labels. Future work includes calibrating the error model to real detector distributions, extending evaluation to hand-drawn circuits, and integrating the pipeline with VLM-based component value extraction.
