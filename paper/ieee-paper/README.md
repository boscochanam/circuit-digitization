# IEEE Access manuscript

Source for:

> **From Hand-Drawn Schematics to SPICE Netlists: A Deterministic Pipeline with Endpoint-Graph
> Wire Joining and a Human-Verified Connectivity Benchmark.**
> Under review at IEEE Access (2026).

The authoritative numbers live in [`docs/research/experiments/SUMMARY.md`](../../docs/research/experiments/SUMMARY.md),
backed by the committed result JSONs in `docs/research/experiments/`. Their provenance — what was
measured, on what data, with which caveats — is recorded in
[`docs/benchmark-provenance.md`](../../docs/benchmark-provenance.md). Do not treat this README as a
source of numbers.

## Two synchronized LaTeX sources

There are two live `.tex` files. They share the **same body**; only the preamble and front matter
differ.

| File | Build | Purpose |
|---|---|---|
| `paper-access.tex` | Overleaf, needs `ieeeaccess.cls` | The submission source |
| `paper-build.tex` | Local `pdflatex`, stock IEEEtran / TeX Live | Local proofing build |

> **Any body edit must be applied to BOTH files.** They are not generated from one another and
> nothing checks that they agree. An edit landed in only one of them silently diverges the proofed
> manuscript from the submitted manuscript.

`paper.tex` is a superseded single-file draft with the old framing and title. It is not built and is
kept only as history. Do not edit it.

## Figures

### Concept diagrams (native TikZ)

`figures/pipeline_overview_tikz.tex`, `figures/endpoint_graph_tikz.tex`,
`figures/completion_tikz.tex`.

The two sources consume these differently:

- `paper-build.tex` `\input`s the `*_tikz.tex` files directly.
- `paper-access.tex` `\includegraphics` the pre-rendered `pipeline_overview.pdf`,
  `endpoint_graph.pdf` and `completion.pdf`, because `ieeeaccess.cls` spot colours conflict with
  TikZ's `\shipout` hook.

Regenerate those PDFs after editing any `*_tikz.tex`:

```bash
./build-tikz-figures.sh
```

### Data figures and image figures

| Figure file | Generated by |
|---|---|
| `figures/wire_benchmark.pdf` | `generate_concept_figures.py` |
| `figures/join_comparison.pdf` | `generate_join_comparison.py` |

`generate_candidates_v2.py` and `generate_join_only.py` are selection aids, not figure builders for
the manuscript: they render candidate panels across many images so a showcase example can be picked
by eye. Both still carry hard-coded absolute paths from the machine they were written on and need
their path constants adjusted before they will run elsewhere.

## Strategy naming

The paper uses descriptive prose names; the code keeps the identifiers. When editing either, keep
this mapping intact.

| Code identifier | Paper prose |
|---|---|
| `scale_completion` | scale-relative graph + completion |
| `degree_budget` | rescue graph + completion |
| `graph_scale` | scale-relative graph (base) |
| `graph_rescue` | rescue graph (base) |
| `production` | radius union-find (legacy) |

`DEFAULT_STRATEGY = "scale_completion"` in `wire_detection/core/join_strategies.py`:
a high-precision scale-relative endpoint-graph base (no end-extension, no dead-end rescue) plus
degree-budget floating-pin completion at reach 4×scale. `degree_budget` and `graph_rescue` remain
registered as fallbacks and ablations.

```bash
# Local proofing PDF

```
